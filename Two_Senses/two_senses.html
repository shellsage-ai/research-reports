🎯 What Is It?

You’re going to build a small wearable gadget that gives you two new “invisible” senses through vibration.
Sense 1 tells you where magnetic north is, and Sense 2 tells you how stale indoor air is (using carbon dioxide levels).
It’s like adding two background instincts your body didn’t have before.

🤔 Why Does It Matter?

Humans already sense light, sound, touch, taste, and smell, but we miss many useful signals around us. This project lets you feel direction and air quality instantly, without staring at a screen.

Personally, this can help you navigate unfamiliar places, notice stuffy rooms sooner, and make better choices about breaks, ventilation, and focus. You’re not just measuring data, you’re training your brain to “feel” hidden information.

#include <Wire.h>
#include "SensirionI2cScd4x.h"
#include "bmm150.h"
#include "bmm150_defs.h"

SensirionI2cScd4x scd4x;
BMM150 bmm = BMM150();

const int VIBE_PIN = 6;
uint16_t co2ppm = 0;
unsigned long lastNorthTick = 0;
unsigned long lastAirAlert = 0;
unsigned long lastCo2Read = 0;

void vibePulse(int msOn, int msOff, int count) {
  for (int i = 0; i < count; i++) {
    digitalWrite(VIBE_PIN, HIGH);
    delay(msOn);
    digitalWrite(VIBE_PIN, LOW);
    if (i < count - 1) delay(msOff);
  }
}

float headingDeg() {
  bmm.read_mag_data();
  float x = bmm.raw_mag_data.raw_datax;
  float y = bmm.raw_mag_data.raw_datay;
  float hdg = atan2(x, y) * 180.0 / PI;
  if (hdg < 0) hdg += 360.0;
  return hdg;
}

float northError(float hdg) {
  float d = fabs(hdg - 0.0);
  return (d > 180.0) ? (360.0 - d) : d;
}

void setup() {
  pinMode(VIBE_PIN, OUTPUT);
  digitalWrite(VIBE_PIN, LOW);
  Serial.begin(115200);
  delay(1000);

  Wire.begin();

  if (bmm.initialize() == BMM150_E_ID_NOT_CONFORM) {
    Serial.println("Compass init failed. Check wiring.");
    while (1) delay(100);
  }

  scd4x.begin(Wire);
  uint16_t error = scd4x.startPeriodicMeasurement();
  if (error) {
    Serial.println("SCD41 init failed. Check wiring.");
    while (1) delay(100);
  }

  Serial.println("Dual-sense wearable started.");
  Serial.println("Compass: face north to feel faster pulses.");
  Serial.println("Air: every 15s, pulses represent CO2 band.");
}

void loop() {
  unsigned long now = millis();

  // Compass update every ~250 ms
  float hdg = headingDeg();
  float err = northError(hdg);

  int interval = 0;
  if (err <= 10) interval = 500;
  else if (err <= 30) interval = 1000;
  else if (err <= 60) interval = 1500;
  else interval = 0;

  if (interval > 0 && (now - lastNorthTick >= (unsigned long)interval)) {
    vibePulse(90, 0, 1);
    lastNorthTick = now;
  }

  // CO2 read every 5 s
  if (now - lastCo2Read >= 5000) {
    bool ready = false;
    if (!scd4x.getDataReadyStatus(ready) && ready) {
      float t, rh;
      scd4x.readMeasurement(co2ppm, t, rh);
      Serial.print("Heading=");
      Serial.print(hdg, 1);
      Serial.print(" deg, Err=");
      Serial.print(err, 1);
      Serial.print(" deg, CO2=");
      Serial.print(co2ppm);
      Serial.println(" ppm");
    }
    lastCo2Read = now;
  }

  // Air quality haptic alert every 15 s
  if (now - lastAirAlert >= 15000) {
    if (co2ppm < 800) vibePulse(60, 0, 1);
    else if (co2ppm < 1000) vibePulse(100, 120, 2);
    else if (co2ppm < 1500) vibePulse(150, 140, 3);
    else vibePulse(250, 150, 4);
    lastAirAlert = now;
  }
}

Phase 4: Calibrate and Use (~10-20 minutes)

  1. Calibrate compass (~2-5 minutes)

    Keep away from laptops, keys, steel desks, and power bricks.

  2. Test the “north sense” (~3 minutes)
    Rotate your body slowly. Vibration should get faster near north and disappear farther away.

  3. Test the “air sense” (~5-10 minutes)
    Breathe near the SCD41 intake briefly, or move into a crowded room. Watch CO2 value rise and pulse pattern change every 15 seconds.

  4. Tune thresholds for your context (~5 minutes)
    Edit these values in code and re-upload:
    800, 1000, 1500 ppm and pulse counts in the if (co2ppm < ...) block.

🌍 Real-World Analogy

Think of I2C sensor wiring like one hallway with labeled apartment doors. The hallway is shared, and each sensor answers only when its address is called. That’s why one microcontroller can talk to many sensors on two data wires.

Think of compass calibration like tuning a guitar before a song. The instrument works out of tune, but calibration brings it into a stable reference so your “north notes” are accurate.

Think of CO2 thresholds like traffic lights. Green means “ventilation is likely okay,” yellow means “getting stuffy,” and red means “open windows or improve airflow now.”

Think of haptic encoding like Morse code for your skin. Different pulse counts and timings carry different meanings without you needing to look at a screen.

⚠️ Safety Note

Do:

  • Keep this as an awareness tool, not a medical safety device.
  • Use vibration feedback while stationary first.
  • Secure wires so they don’t snag while walking.

Don’t:

  • Use it as your only navigation aid in risky environments.
  • Assume CO2 alone defines all indoor air hazards.
  • Wear loose wiring near moving machinery or bike chains.

❌ Common Mistakes & How to Avoid Them

  1. Nothing uploads to board
    What goes wrong: No device found on COMx.
    Why: Wrong port or charge-only cable.
    Fix: Run arduino-cli board list and use a known data cable.

  2. CO2 always reads zero or stale values
    What goes wrong: No meaningful CO2 updates.
    Why: Periodic measurement not started, or not waiting long enough.
    Fix: Ensure startPeriodicMeasurement() runs and wait at least 5 seconds per read.

  3. Compass heading jumps wildly
    What goes wrong: North vibration feels random.
    Why: Magnetic interference from metal/electronics nearby.
    Fix: Calibrate with figure-8 and keep sensor away from magnets/steel surfaces.

  4. Both I2C sensors don’t show up together
    What goes wrong: One sensor works, other fails.
    Why: Incorrect hub wiring or loose connector.
    Fix: Re-seat all Grove cables and use the Grove - I2C Hub for branching.

  5. Vibration is too weak to notice
    What goes wrong: You miss cues while moving.
    Why: Motor not touching body firmly or pulse too short.
    Fix: Increase pulse duration (e.g., 90 to 150 ms) and mount motor against skin/clothing.

  6. Reader interprets CO2 as direct toxicity warning
    What goes wrong: Wrong decisions from misunderstood data.
    Why: CO2 is mostly a ventilation proxy at common indoor levels.
    Fix: Use it to guide ventilation habits, not as a complete air safety diagnosis.

🔬 Going Deeper

Earth’s magnetic field is usually around 25,000 to 65,000 nT (that is 25-65 µT), which is right in the usable range for compact compass sensors. Your BMM150-based module supports approximately ±1300 µT (x/y) and ±2500 µT (z), so Earth’s field is comfortably measurable.

The SCD4x family reports a broad CO2 output range, while best accuracy is specified in the typical indoor decision range. For SCD41, a commonly cited accuracy spec is ±(40 ppm + 5% of reading), which is why it’s good for trend-based decisions (ventilate sooner/later), not lab-grade gas certification.

A practical interpretation ladder:

  • ~400-600 ppm: near outdoor baseline or well-ventilated indoors.
  • ~600-1000 ppm: commonly acceptable indoor range.
  • >1000 ppm: often indicates under-ventilation; comfort/cognitive complaints may increase.
  • 5000 ppm (occupational 8h limit context): far above normal indoor comfort targets.

Your coding cadence matters:

  • Compass feedback every 250 ms feels responsive.
  • CO2 updates every 5 s match sensor behavior.
  • Air alerts every 15 s avoid overwhelming your attention.

This timing ratio is intentional: fast channel for orientation, slower channel for environmental context.

📖 Glossary

  • Augmentation: Adding new abilities to your body using tools or devices.
  • Compass heading: Direction in degrees from 0 to 360, where 0 is north.
  • CO2 (carbon dioxide): A gas in air; indoors, it often tracks ventilation quality.
  • I2C (Inter-Integrated Circuit): A two-wire communication method for multiple sensors.
  • Microcontroller: A tiny programmable computer that controls sensors and outputs.
  • NDIR (non-dispersive infrared): A gas-sensing method used to measure true CO2.
  • ppm (parts per million): Unit for tiny concentrations, like CO2 in air.
  • Calibration: Adjusting sensor behavior so readings are stable and meaningful.
  • Haptic feedback: Information delivered through touch, usually vibration.
  • Threshold: A chosen boundary value that triggers an action.
  • Serial monitor: Text console showing live data from your board.
  • Geomagnetic field: Earth’s magnetic field used by compasses for direction.
