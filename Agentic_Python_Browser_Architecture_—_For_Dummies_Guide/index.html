Agentic Python Browser Architecture

🎯 What Is It?

An agentic browser is a browser that can decide what to do next, not just follow one hard-coded script.
You give it a goal like “find the latest NASA headline,” and it loops: look at page, think, click/type, repeat.
In this guide, you will build one in Python that actually runs in a real Chromium browser window.

🤔 Why Does It Matter?

Most automation breaks when one button moves. Agentic browsing is more flexible because the model can adapt step-by-step.

This matters personally if you do repetitive web tasks: form filling, research collection, checking dashboards, or pulling structured info from websites. Once you build this, you can reuse the same “agent brain” for many tasks instead of writing a new script every time.

import json
import os
import re
import sys
import time
from typing import Any, Dict, List, Tuple

from dotenv import load_dotenv
from openai import OpenAI
from playwright.sync_api import sync_playwright


SYSTEM_PROMPT = """
You are a browser agent.
Return ONLY valid JSON with this shape:
{
  "action": "goto|click|type|wait|done",
  "element_id": 0,
  "text": "",
  "url": "",
  "seconds": 1,
  "answer": "",
  "reason": ""
}

Rules:
- Use only listed element_id values.
- Prefer short, safe actions.
- If goal is complete, return action="done" with a clear answer.
- If page still loading, use wait.
"""

MAX_STEPS = 15


def extract_json(text: str) -> Dict[str, Any]:
    text = text.strip()
    try:
        return json.loads(text)
    except Exception:
        match = re.search(r"\{.*\}", text, re.DOTALL)
        if not match:
            raise ValueError("No JSON found in model output.")
        return json.loads(match.group(0))


def get_interactables(page) -> List[Dict[str, Any]]:
    js = r"""
    () => {
      function cssPath(el) {
        if (!el || !el.tagName) return "";
        if (el.id) return "#" + CSS.escape(el.id);
        const parts = [];
        let cur = el;
        let depth = 0;
        while (cur && cur.nodeType === 1 && depth < 6) {
          let part = cur.tagName.toLowerCase();
          const parent = cur.parentElement;
          if (parent) {
            const siblings = Array.from(parent.children).filter(
              x => x.tagName === cur.tagName
            );
            if (siblings.length > 1) {
              part += `:nth-of-type(${siblings.indexOf(cur) + 1})`;
            }
          }
          parts.unshift(part);
          cur = parent;
          depth++;
        }
        return parts.join(" > ");
      }

      const candidates = Array.from(
        document.querySelectorAll("a,button,input,textarea,select,[role='button']")
      );

      const visible = candidates.filter(el => {
        const r = el.getBoundingClientRect();
        const style = window.getComputedStyle(el);
        return r.width > 0 && r.height > 0 && style.visibility !== "hidden" && style.display !== "none";
      });

      return visible.slice(0, 40).map((el, idx) => ({
        id: idx,
        tag: (el.tagName || "").toLowerCase(),
        text: (el.innerText || el.textContent || "").trim().slice(0, 80),
        aria_label: (el.getAttribute("aria-label") || "").slice(0, 80),
        placeholder: (el.getAttribute("placeholder") || "").slice(0, 80),
        href: (el.getAttribute("href") || "").slice(0, 120),
        selector: cssPath(el),
        can_type: ["input", "textarea"].includes((el.tagName || "").toLowerCase())
      }));
    }
    """
    return page.evaluate(js)


def run_action(page, action: Dict[str, Any], elements: List[Dict[str, Any]]) -> Tuple[bool, str]:
    name = action.get("action", "").strip().lower()

    if name == "goto":
        url = action.get("url", "").strip()
        if not (url.startswith("http://") or url.startswith("https://")):
            return False, f"Invalid URL: {url}"
        page.goto(url, wait_until="domcontentloaded", timeout=20000)
        return True, f"Navigated to {url}"

    if name == "click":
        idx = int(action.get("element_id", -1))
        if idx < 0 or idx >= len(elements):
            return False, f"Bad element_id: {idx}"
        selector = elements[idx]["selector"]
        page.locator(selector).first.click(timeout=10000)
        page.wait_for_load_state("domcontentloaded", timeout=10000)
        return True, f"Clicked element {idx}: {selector}"

    if name == "type":
        idx = int(action.get("element_id", -1))
        text = action.get("text", "")
        if idx < 0 or idx >= len(elements):
            return False, f"Bad element_id: {idx}"
        selector = elements[idx]["selector"]
        loc = page.locator(selector).first
        loc.click(timeout=10000)
        loc.fill(str(text), timeout=10000)
        if action.get("press_enter", True):
            loc.press("Enter")
            page.wait_for_load_state("domcontentloaded", timeout=10000)
        return True, f"Typed into element {idx}: {selector}"

    if name == "wait":
        sec = float(action.get("seconds", 1))
        sec = max(0.5, min(sec, 10))
        time.sleep(sec)
        return True, f"Waited {sec:.1f}s"

    if name == "done":
        return True, f"DONE: {action.get('answer', '').strip()}"

    return False, f"Unknown action: {name}"


def main() -> None:
    load_dotenv()
    api_key = os.getenv("OPENAI_API_KEY")
    model = os.getenv("MODEL", "gpt-5-mini")

    if not api_key:
        raise RuntimeError("OPENAI_API_KEY not found in .env")

    if len(sys.argv) < 2:
        print('Usage: python .\\agentic_browser.py "your goal here"')
        sys.exit(1)

    goal = " ".join(sys.argv[1:])
    client = OpenAI(api_key=api_key)

    with sync_playwright() as p:
        browser = p.chromium.launch(headless=False, slow_mo=250)
        context = browser.new_context(viewport={"width": 1280, "height": 800})
        page = context.new_page()
        page.goto("https://example.com", wait_until="domcontentloaded")

        history: List[Dict[str, Any]] = []

        for step in range(1, MAX_STEPS + 1):
            elements = get_interactables(page)
            state = {
                "goal": goal,
                "step": step,
                "max_steps": MAX_STEPS,
                "url": page.url,
                "title": page.title(),
                "elements": [
                    {
                        "id": e["id"],
                        "tag": e["tag"],
                        "text": e["text"],
                        "aria_label": e["aria_label"],
                        "placeholder": e["placeholder"],
                        "href": e["href"],
                        "can_type": e["can_type"],
                    }
                    for e in elements
                ],
                "recent_history": history[-5:],
            }

            response = client.responses.create(
                model=model,
                input=[
                    {"role": "system", "content": SYSTEM_PROMPT},
                    {"role": "user", "content": json.dumps(state, ensure_ascii=False)},
                ],
            )

            raw = (response.output_text or "").strip()
            action = extract_json(raw)

            ok, result = run_action(page, action, elements)
            history.append({"step": step, "action": action, "result": result})

            print(f"[Step {step}] {action.get('action')} -> {result}")

            if action.get("action", "").lower() == "done":
                print("\nFinal answer:")
                print(action.get("answer", ""))
                break

            if not ok:
                print("Stopping due to action error.")
                break
        else:
            print("\nReached step limit. Increase MAX_STEPS if needed.")

        context.close()
        browser.close()


if __name__ == "__main__":
    main()

Phase 3: Improve Reliability (~10-20 minutes)

  1. Use Playwright code generator to discover robust selectors (~5 minutes)
    Run:
playwright codegen https://example.com

This helps you inspect stable interaction targets.
⚠️ Don’t: rely only on fragile CSS class names from dynamic sites.

  1. Lower cost during testing (~2 minutes)
    In .env, keep:
MODEL=gpt-5-mini

Then rerun your script. Smaller models are usually cheaper for iteration.

  1. Add a safety stop for dangerous actions (~3 minutes)
    Keep MAX_STEPS = 15 (or lower) in code.
    ⚠️ Don’t: run unlimited loops on unknown sites.

⚠️ Safety Note

  • Do: test first on harmless sites like https://example.com or docs pages.
  • Do: keep headless=False while learning so you can watch actions.
  • Don’t: store API keys in code files or screenshots. Use .env.
  • Don’t: automate login to banking/medical/government portals until you add strict action allowlists and domain restrictions.
  • Don’t: bypass captchas or terms of service of websites.

🌍 Real-World Analogy

Think of the agent loop like a self-driving delivery bike: it checks the road (page state), decides the next turn (model output), then moves one block (action).

Think of Playwright like robotic hands and eyes: it can see buttons and type in boxes, but it needs instructions.

Think of the model like a navigator: it doesn’t touch the steering wheel directly; it tells the driver which move to make.

Think of the action schema (JSON) like a restaurant order ticket: short, structured, and unambiguous so the kitchen (your Python code) can execute correctly.

Think of step limits and guardrails like a leash on an excited dog: it still explores, but cannot run forever or into unsafe places.

❌ Common Mistakes & How to Avoid Them

  1. Agent says “browser executable not found.”
    What goes wrong: Playwright can’t launch Chromium.
    Why: you installed package but not browser binaries.
    Fix: run python -m playwright install chromium.

  2. OPENAI_API_KEY not found error.
    What goes wrong: model call fails immediately.
    Why: .env missing or wrong directory.
    Fix: keep .env in C:\agentic-browser\ and activate venv before running.

  3. Agent clicks wrong thing repeatedly.
    What goes wrong: loop burns steps with no progress.
    Why: page elements are ambiguous or hidden.
    Fix: reduce candidate list, add clearer prompt rules, and include visible text + labels.

  4. Script works once, then fails on another site.
    What goes wrong: fragile selectors break.
    Why: sites have dynamic structure/classes.
    Fix: prioritize role/text logic and use playwright codegen for stable patterns.

  5. Costs grow unexpectedly.
    What goes wrong: many model calls per run.
    Why: high step count and verbose state payload.
    Fix: lower MAX_STEPS, shorten page snapshot text, use lower-cost model for testing.

  6. PowerShell activation blocked.
    What goes wrong: .venv activation fails.
    Why: execution policy restriction.
    Fix: run Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser.

✅ Key Takeaways

  • Agentic browser = observe, decide, act, repeat.
  • Playwright controls the browser; model controls decisions.
  • Keep API keys in .env, never in source code.
  • Start with headless=False for visible debugging.
  • Limit steps to control risk and cost.
  • Use structured JSON actions for reliability.
  • Test on simple websites before real tasks.

🔬 Going Deeper

Playwright docs list Python 3.8+ support and install flow (pip install playwright + playwright install). That split matters: one command installs Python code, the other installs browser engines.

Your agent run cost is mostly proportional to:

  • number_of_steps
  • state_size_per_step (how much page data you send)
  • model_price_per_token

As of March 6, 2026, OpenAI pricing pages show token-based billing and multiple tiers. A practical beginner strategy is:

  • Start with a lower-cost model for debugging.
  • Trim state to top 20-40 visible elements.
  • Keep steps at 10-15 until behavior is stable.

Reliability usually improves when you constrain outputs. Requiring strict JSON action objects reduces “chatty” responses and makes error handling deterministic. In real projects, you can go further by using formal tool-calling interfaces and domain allowlists (for example, only allow https://docs.python.org during a documentation task).

📖 Glossary

  • Agentic: able to choose next actions toward a goal.
  • Playwright: Python library that automates browsers.
  • Model: the language system that decides what to do next.
  • Application Programming Interface (API): a way software talks to another service.
  • API key: secret token proving your app can use an API.
  • Virtual environment (venv): isolated Python package workspace per project.
  • Chromium: open-source browser engine used by Playwright here.
  • Selector: rule for finding a page element (button, input, link).
  • JSON (JavaScript Object Notation): structured text format for data.
  • Guardrail: rule that limits unsafe or runaway behavior.
  • Headless mode: browser runs without visible window.
  • Token: chunk of text used for model input/output billing.
